# z-index를 알아보자!

## what is z-index?
`z-index`는 HTML element의 z축 즉 겹치는 순서를 나타낸다.
`z-index`의 값으로는 integer 또는 auto를 가질수 있다.
integer로 설정한 경우 높을 수록 위에 위치한다.
auto로 설정한 경우 부모로 부터 `z-index`값을 상속받는다.
그러나 가끔식 `z-index`를 아무리 낮게 혹은 높게 설정해도 원하는 대로 동작하지 않는 경우가 있다.
왜 이런 일이 발생하는지 알아보자!

## default stacking order


## z-index는 position: static;로 설정된 경우 동작하지 않는다.
position 을 갖고 있는 요소는 그것을 가지고 있지 않는 요소보다 앞에서 노출

`position: static;`은 position속성의 default값이다.
이 경우 `z-index`는 무시된다.

## stacking context
`z-index`는 stacking context안에서 동작한다.
서로 다른 stacking context에 있는 element에 아무리 높은 `z-index`값을 줘봤자 다른 stacking context에 있는 element에 영향을 미치지 못한다.
그렇다면 stacking context는 언제 생성될까?


## stacking context는 어떻게 생성되는가?
* `<html>`즉 HTML의 root element는 stacking context생성한다.
* `z-index: <integer>`로 설정한 경우
* `position: absolute | relative` 이면서 동시에 `z-index: <integer>`를 설정한 경우
